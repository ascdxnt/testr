# Simulation: My First Test Run

**Course:** SC-405 Software Quality
**Author:** Jeferson Andrew Fuentes García
**Section:** LN

## 1. Project description

This project implements a suite of automated functional tests using the
**Page Object Model** pattern, with **Selenium WebDriver** for browser
interaction, **TestNG** as the structuring and execution framework, and
**Extent Reports** for execution evidence generation.

## 2. Application under test

The application under test is **Ticketr**, an event ticket sales and
management system built with Spring Boot, Thymeleaf, and MySQL. It is a
personal project, never used during the course's class sessions, in
compliance with the assignment's requirement to freely choose a web
application.

## 3. Technologies used

| Technology | Version | Purpose |
|---|---|---|
| Java | 17+ | Implementation language |
| Maven | 3.9 | Dependency management and project lifecycle |
| Selenium WebDriver | 4.34.0 | Browser automation |
| TestNG | 7.11.0 | Test case structuring and execution |
| WebDriverManager | 6.3.0 | Automatic Chrome driver management |
| Extent Reports | 5.1.2 | Execution report generation |

## 4. Project structure

```
pom.xml                             Maven configuration and dependencies
src/main/java/pages/                Page Objects (one per Ticketr screen)
src/test/java/tests/                Base class and TestNG test cases
src/test/java/utils/                Configuration and reporting utilities
src/test/resources/                 Environment configuration and TestNG suite
reports/report.html                 Execution report generated by Extent Reports
screenshots/                        Photographic evidence captured during execution
Test-Case-Template-Ticketr.xlsx     Test case documentation
```

## 5. Navigation scenario and test cases

The suite walks through five distinct screens of the application
(Registration, Login, Events listing, Event detail, and My Purchases),
split across four independent test cases, each with its own assertions:

| ID | Test case | Screens covered |
|---|---|---|
| TC01 | Register a new user | Registration → Login |
| TC02 | Administrator login | Login → Events (listing) |
| TC03 | View event detail and purchase tickets | Events (listing) → Event (detail) → Events (confirmation) |
| TC04 | Verify purchase history | Events (listing) → Event (detail) → My Purchases |

Preconditions, steps, and expected results for each case are documented in
`Test-Case-Template-Ticketr.xlsx`.

## 6. Execution requirements

- Java 17 or higher, and Maven.
- Google Chrome installed.
- A running MySQL server with the `sistema_boletos` database created.
- The Ticketr application running at `http://localhost:8080`.
- At least one event registered with available tickets (required by TC03
  and TC04).

## 7. Running the suite

```bash
mvn test
```

This runs the suite defined in `src/test/resources/testng.xml`. It can also
be run from any IDE with Maven and TestNG support.

## 8. Results

The suite was run in full against a live Ticketr instance, with all four
test cases passing (`4/4 passed`). The execution report, including logs,
per-case timing metrics, and photographic evidence, is available at
`reports/report.html`.
